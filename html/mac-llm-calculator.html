สิ่งที่ ควรรู้ สำหรับ Mac + LLM
01
Unified Memory = VRAM แบบ Flex
Mac ไม่มีขีดจำกัดแยก VRAM กับ RAM — Memory ทั้งหมดพร้อมใช้สำหรับ LLM ทำให้ Mac 64 GB รัน 70B ได้ทันทีโดยไม่ต้อง CPU offload
02
Ollama คือเครื่องมือแนะนำสุด
ติดตั้งง่ายที่สุด รัน ollama run llama3 แล้วได้เลย รองรับ Metal อัตโนมัติ LM Studio ก็ดี มี GUI สวย ใช้ง่ายสำหรับ Beginner
03
MLX เร็วกว่า llama.cpp บน Mac
Apple's MLX framework ออกแบบมาสำหรับ Apple Silicon โดยตรง ให้ token speed สูงกว่า llama.cpp ประมาณ 10–30% ลอง mlx-lm บน Python
04
Bandwidth คือปัจจัยความเร็ว
M4 Max (546 GB/s) เร็วกว่า M4 base (120 GB/s) เกือบ 5 เท่าสำหรับ LLM ทั้งที่ compute score ต่างกันน้อย เลือก chip ที่ bandwidth สูงถ้า priority คือความเร็ว
05
เหลือ Memory ให้ระบบด้วย
macOS ใช้ Memory ไป 3–6 GB ขึ้นอยู่กับ Apps ที่เปิด แนะนำเหลือ buffer อย่างน้อย 4 GB เช่น Mac 16 GB จริงๆ ใช้รัน LLM ได้สบายๆ แค่ ~12 GB
06
เทียบ RTX: แต่ละ use-case ชนะต่างกัน
7B–13B: RTX 4090 เร็วกว่า 2–3× · 30B+: ขึ้นกับ VRAM ที่มี · 70B+: Mac ชนะชัดเจน (RTX ไม่พอ VRAM) · พกพา + เงียบ + ไฟน้อย: Mac ชนะทุกกรณี
